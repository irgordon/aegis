# AEGIS v0.4.0 Release Notes

## Release Identity

AEGIS v0.4.0 is the Minimum Usable Local Release.

It is local-only, pre-alpha, developer-oriented, and not production-ready.

It is not enterprise-hardened.

## Summary

This release candidate proves that AEGIS can run a local governed execution path and show operator evidence in a desktop UI.

The release is meant for local evaluation from source. It is not meant to protect real systems.

## What This Release Includes

- Tauri plus Slint desktop shell.
- v0.4.0 visual design system.
- Visual readability review evidence.
- Read-only live backend `health.check` evidence through fixed IPC.
- Sample evidence rendering for execution, recovery, planning, and normalized error surfaces.
- Local gateway execution.
- Built-in `health.check` wrapper.
- Built-in `sandbox.note.write` wrapper.
- Policy bundle verification with checksum and Ed25519 signature verification.
- Local policy evaluation.
- Execution authorization boundary.
- Credential boundary.
- Local credential handle boundary.
- Wrapper dispatch.
- Audit JSONL evidence.
- State JSONL evidence.
- Recovery inspection.
- Recovery planning.
- Structured normalized errors.
- Executable v0.4.0 release validation script.

## What Users Can Do

- Launch the desktop app locally.
- View the operator surface.
- See live health-check evidence.
- Run local gateway smoke checks.
- Run `sandbox.note.write` in a controlled local sandbox.
- Inspect generated state evidence.
- Generate a recovery plan from state evidence.
- Run the v0.4.0 release validation gate.
- Review release limitations.

## What This Release Does Not Include

- Production readiness.
- Enterprise hardening.
- Installer packages.
- Code signing.
- Auto-update.
- Release artifacts.
- Approval workflow.
- Replay execution.
- Recovery execution.
- Production credential providers.
- Vault integration.
- Cloud identity.
- HTTP API.
- Service deployment.
- Database persistence.
- Remote logging.
- SIEM export.
- External integrations.
- Multi-user operation.
- Plugin architecture.
- Arbitrary UI-driven gateway execution.
- UI-driven mutation execution.
- UI-driven policy selection.
- UI-driven request file selection.
- UI-driven audit or state path selection.

## Validation Status

The v0.4.0 release validation gate passes.

The gate validates repository governance, Rust checks, desktop checks, UI scaffold tests, gateway smoke tests, recovery inspection, recovery planning, desktop launch, and repository cleanliness.

## Visual Readability Status

Visual readability review passed.

Evidence is recorded in `docs/assets/release/v0.4.0-desktop-readability-review.md`.

## Security Notes

- UI remains an operator surface, not an authority boundary.
- Read-only IPC is limited to fixed health-check evidence.
- Mutation execution remains outside the UI.
- Recovery inspection and recovery planning remain read-only.
- No production credential providers, vault integration, cloud identity, approval workflow, replay execution, HTTP service, or external integrations are included.

## Known Limitations

- Users must build and run from source.
- Desktop UI live evidence is limited to fixed `health.check` evidence.
- Recovery cards still show labeled sample evidence.
- Audit and state logs are local JSONL development evidence.
- Local credential handles are safe references, not real credentials.
- Packaging, signing, publishing, and installer generation are not included.
